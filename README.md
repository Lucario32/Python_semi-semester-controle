# Описание алгоритма

Алгоритм ищет аномально высокую активность респондентов по отдельным брендам в каждый день.

*Логика:*
- Рассчитывается `daily_ots = Weight × количество запросов` для каждой комбинации (респондент + день + бренд).
- Для каждого бренда и дня отдельно считается Robust Z-score.
- Если Z-score > 3.5 — респондент признаётся аномальным за этот день.
- Удаляются *все* записи респондента за этот день.

*Почему этот подход:*
- Интерпретируемый (есть понятный score).
- Работает на уровне брендов.
- Устойчив к выбросам и малым выборкам.
- Соответствует статистическим методам из статей по anomaly detection.
